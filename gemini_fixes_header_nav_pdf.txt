PROBLEM A1 — Header bar is too tall in landscape mode

ROOT CAUSE:
  Line 889: `py-4` gives the header 32px vertical padding (16px top + 16px bottom)
  plus text height (~28px title + ~14px subtitle) = total ~74px header.
  On a landscape phone (~375px height), this header wastes 20% of the screen.
  Additionally the exam title uses `text-lg` (18px) — too large for landscape.

  - py-4 → py-2: reduces vertical padding from 32px to 16px on all screens.

PROBLEM B1 — Share button in error modal sends the APP link, not a download link
────────────────────────────────────────────────────────────────────────────────

ROOT CAUSE:
  When a student uses an in-app browser (WhatsApp/Instagram) or when iOS
  Safari blocks the popup, the download fails and a warning modal appears.

  In `handleDownloadUrl` (Oracle path):

  CASE 1 — In-app browser (lines ~288-295):
    setDownloadMessage({ title: '⚠️ Browser Not Supported', isWarning: true })
    → NO `fallbackUrl` or `item` is passed.
    → Modal only shows "Copy App Link" and "WhatsApp App Link" buttons.
    → Student who clicks WhatsApp button gets the APP URL, not the PDF.
    → When recipient opens the app URL in WhatsApp browser, they STILL can't
       download. Infinite loop. This is the bug the user reported.

  CASE 2 — iOS popup blocked (lines ~300-309):
    setDownloadMessage({ title: '⚠️ Popup Blocked', isWarning: true })
    → Same problem: NO `fallbackUrl` or `item` passed.
    → No Oracle direct link is offered.

  THE FIX: Pass `fallbackUrl: item.contentUrl` and `item: item` to the
  downloadMessage in BOTH cases. This makes the modal show the Oracle direct
  download link button AND the "WhatsApp Oracle Link" button — both of which
  allow the student to get the actual PDF without needing the app.

PROBLEM B3 — PDF password inconsistency between Oracle and Chunked paths
────────────────────────────────────────────────────────────────────────────────

ROOT CAUSE:
  handleDownloadUrl (Oracle): uses `cleanPhone(rawPhone)` → always 10 digits.
    Example: "+919876543210" → "9876543210"

  handleDownloadChunked: uses its own logic on line ~404:
